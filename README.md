## Core Gameplay

Игрок должен анализировать аномалии в окружении и определять правильную дверь.

Аномалии не дают прямого ответа, а выступают как косвенные подсказки, требующие интерпретации.

Дополнительно реализуется система анализа аномалий через интерфейс:
- игрок собирает информацию об аномалиях
- может анализировать их в инвентаре
- каждая аномалия содержит подсказку о правильном выборе

Без анализа вероятность пройти игру минимальна.

Геймплей строится не на реакции, а на анализе и принятии решений.

## Архитектура системы

Проект построен вокруг централизованной системы управления комнатой (Room), выступает как центральная система управления:

- инициализацией систем
- спавном объектов
- применением аномалий
- логикой дверей

Дополнительно используются менеджеры:

- AnomalyManager — определяет аномалии для комнаты
- SessionManager — управляет прогрессией и переходами
- RoomManager — хранит текущую комнату
- отдельные менеджеры для подсистем (загадки, звук, взаимодействие)

На текущем этапе используется синглтон-подход для упрощения взаимодействия между системами.

---

## Основные системы

### Room System
- управление состоянием комнаты
- контроль порядка инициализации
- координация всех систем

---

### Anomaly System
- динамическое применение аномалий
- выбор аномалий через AnomalyManager
- fallback система (SoundHints) при ошибках

---

### Door System
- выбор правильной двери
- изменение логики в зависимости от аномалии
- пример: ClockPuzzle (логика зависит от времени на часах)

---

### Puzzle Systems
- часы (влияние на выбор двери)
- записки
- UV flashlight система

---

### Voice Guide System
Система голосовых подсказок:

- очередь сообщений
- приоритетные сообщения
- эффект печати текста
- аудио сопровождение

Используется для передачи информации игроку и создания атмосферы.

---

### Session System
Система прогрессии:

- переход между сессиями
- сброс состояния игры
- управление UI
- взаимодействие с системой аномалий

---

### Sanity System
Система психического состояния игрока:

- снижение sanity при ошибках
- накопительный эффект (усиление штрафа при повторных ошибках)
- несколько стадий состояния (Normal → Stage1 → Stage2 → Stage3)

Система влияет на восприятие игрока:
- визуальные эффекты (vignette, chromatic aberration, grain)
- изменение цветового восприятия
- усиление давления при ошибках

Таким образом, система напрямую влияет на способность игрока анализировать аномалии.

## Технические решения

- централизованная система управления
- строгий порядок инициализации:
  менеджеры → объекты → двери → аномалии
- зависимость геймплея от состояния объектов (например часы)
- использование fallback логики для устойчивости системы
- разделение ответственности между системами

---

## Проблемы и решения

Проблема: рост количества префабов  
Решение: одна комната + процедурная вариативность  

Проблема: высокая связность систем  
Решение: контроль порядка инициализации  

Проблема: ошибки в логике аномалий  
Решение: fallback система
